go基础之数组

我认为学习一门语言重要的不是去记一些，如何定义数组，数组什么形式，…型定义，这些语法的东西，因为计算机语言跟我们用来沟通的语言有很多相似之处，语言的本质是用来交流，我们学习英文会记很多语法，但最后发现也没有因为我们记了这么多语法就能更好的和外国人交流。舍本逐末不仅浪费时间，还消耗精力。我认为更多的时间应该应该放在理解更底层的东西

知识点概括

  1. 数组是固定长度的，在go语言中我们一般用slice代替数组，因为slice可以动态的改变长度。
  2. 数组所有元素的类型必须相同。
  3. 数组的0值，就是对应数组每个元素的0值。
  4. 数组的定义：
    1. var q [3]int = [3]int{1, 2, 3}
    2. q := […]int{1, 2, 3}
    3. a:=[3]int{1,2,3}

注意，在类型判断中，区别array和slice的唯一区别就是array是[3]int,slice是[]int

  1. 数组由两部分组成：长度和数据类型，所以判断两个数组是否相等，一定是长度和数据类型都相等的情况下，每个元素分别相等。
  2. 数组也可以像struct那样，字面量初始化。把数组看成是一个

array{ index int obj object }

example := […]string{1: “$”, 2: “€”, 3: “￡”, 4: “￥”}

这里，0的下标为空string。还比如：r := […]int{99: -1}，定义了一个含有100个元素的数组r，最后一个元素被初始化为-1，其它元素都是用0初始化。 7. 数组的创建，赋值，传递，都是值的复制。

a:=[3]int{1,2,3}

b:=a

a[1]=100

fmt.Printf("a= %v\n",a)
fmt.Printf("b= %v\n",b)

// output:
// a= [1,100,3]
// b= [1,2,3]

从数组到slice

go语言肯定不是平白无故搞出来一个slice，语言的语法糖的初衷肯定是想要方便我们开发者的，比如es6里那么多好用的语法糖。所以，看到一个新的用法，我们要开始感兴趣，而不是觉得麻烦，不想去学习新东西，因为用起来很可能会提升你的开发效率，事半功倍。

数组的问题1：值传递

从以上我们可能已经发现，基础知识第七点，既然数组的传递是值传递，值的话，内存占用有大也有小，那如果传递一个较大（这里的大是内存占用较大）的值，就会产生一些效率的问题。比如，我有一个1000万长度的数组，要传递给一个方法去处理，先不说处理这1000万个值的效率，就传递这个值就很费劲。这也是slice出现的最大诱因之一。在抛开语言基础的情况下，解决值传递内存问题的方法就是传递指针或者叫引用。

在其他语言中，我们看到的可能是直接传递数组，其实这是一种传递引用的隐式实现。底层也是在传递一个指针。

数组的问题2：长度固定

长度固定在内存方面来说，便于管理和优化。但是在应用中，必然不是一个方便的设计，因为我们在创建一个数组的时候，可能根本不知道他的长度，还有就是长度也是数组类型的一部分。

func get(a [3]int){
    fmt.Println(a)
}

这时候你就不能传{1,2,3,4}这个数组，因为长度要求必须是3，显然这很不方便。

slice应用而生

slice主要解决以上两个问题

  1. slice的传递，赋值等都是引用（指针）传递
  2. 长度可变
